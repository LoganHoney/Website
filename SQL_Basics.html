Your guide to mastering the fundamentals of SQL

A Brief History of SQL

SQL, which stands for Structured Query Language, was developed in the early 1970s by Donald D. Chamberlin and Raymond F. Boyce at IBM. It was originally called SEQUEL and used to query IBM’s System R database.

By 1986, SQL became the standard for interacting with relational databases, and today it powers systems like MySQL, PostgreSQL, Oracle, and SQL Server.

What Is a Relational Database?

A relational database organizes data into tables made of rows and columns. Each table is like a spreadsheet, where:

  • Columns define data types (e.g., Name, Email)
  • Rows hold actual data records

Tables can be connected using keys, such as primary and foreign keys.

Example:

Customers Table

Customer_ID | First_Name | Last_Name
----------- | -----------|----------
101         | Alice      | Johnson
102         | Bob        | Smith

Orders Table

Order_ID | Customer_ID | Product
-------- | ------------|--------
001      | 101         | Laptop
002      | 102         | Phone

This structure allows us to connect which customer made which order using the Customer_ID.

SQL Syntax Basics: SELECT, FROM, WHERE

Let’s explore the core SQL statements used to retrieve data:

1. SELECT – Choose the data

SELECT First_Name, Last_Name

2. FROM – Specify the table

FROM Customers

3. WHERE – Filter the results

WHERE Customer_ID = 101;

Full Example:

SELECT First_Name, Last_Name
FROM Customers
WHERE Customer_ID = 101;

This query returns the name of the customer whose ID is 101.

Wrapping Up

SQL is the foundation of working with structured data. Whether you're analyzing business trends or building applications, knowing how to query data is an essential skill.
